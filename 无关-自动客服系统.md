# 自动客服 Agent

## 思路

两个 Agent，一离线、一在线：

| Agent | 职责 |
|-------|------|
| **蒸馏** | 把 PDF、网页、FAQ 等原始资料整理成结构化 MD 知识库 |
| **问答** | 用户提问 → 检索相关 MD 片段 → 基于片段生成回答 |

## 流程

```
[离线] 原始资料 → 蒸馏 Agent → MD 知识库
[在线] 用户问题 → 检索 MD → 问答 Agent → 回复
```

## 要点

- MD 按主题分篇，带清晰标题，方便切块检索
- 检索不到或不确定 → 转人工，不编造
- 资料更新 → 重新蒸馏 → 更新知识库

## 两个细节

**蒸馏：全量还是 patch？**

- 首次：原始资料 → 全量生成 MD
- 更新：用 patch 只改变动章节（保留人工改过的内容）；小项目也可全量重跑

**问答：全检索还是片段？**

- 片段。按标题切块，检索 Top-K 相关段 → 只把这些段给 LLM 回答
- 不把整份 MD 塞进上下文（知识库一大就装不下、也易跑偏）
